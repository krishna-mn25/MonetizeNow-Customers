Proposed intake process — deletion request

1
End customer submits erasure request to Mercell
Customer invokes right to erasure (GDPR Art. 17). Mercell logs: customer email, request date, internal request ID.
Mercell
2
Mercell submits deletion request to MN via agreed channel
Via Slack or Pylon ticket with: (a) contact email, (b) account name, (c) request date, (d) Mercell internal request ID. Batch submissions preferred.
Mercell → MN (shared action)
3
MN confirms scope before any execution
MN confirms what will be deleted (contact PII) vs. retained (financial records). Mercell DPO confirms. No execution without explicit sign-off.

4
MN executes deletion within 7 working days
Batch processing on Monday and Thursday. Sub-processor notification (DocuSign / Adobe Sign) initiated where applicable.
MN
5
MN sends written confirmation to Mercell S&RevOps + DPO
Confirmation includes: what was deleted, what was retained and why, execution date, MN reference ID. Mercell DPO uses this to close the customer request within their 30-day window.
