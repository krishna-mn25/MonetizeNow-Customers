Trials

Trials give customers a defined window to evaluate products before any billing starts. MonetizeNow manages the full trial lifecycle: creation, extension, conversion to a paid subscription, and cancellation.

What is a trial?

A trial is a standalone record in MonetizeNow, separate from quotes and subscriptions. It holds everything needed to become a paid subscription, but nothing about billing runs until conversion happens. Trials are always free. Billing starts from the contract start date on the subscription created at conversion.

When you create a trial, you configure two things up front:

  • What the paid subscription will look like - contract length, net terms, and contacts for the resulting quote
  • What products and pricing to convert onto - the offerings, rates, and quantities that go on the converted quote

Both are locked in at creation. When the trial converts, MonetizeNow replays that configuration automatically without needing any additional input.

How a trial ends

Every trial is created with an endAction. This controls what happens automatically when the trial period expires.

End action What happens at the end date Default
CONVERT_TO_PAID MonetizeNow creates a fully processed quote and starts the subscription automatically. No manual step required.
CANCEL Trial expires. No quote is created, no subscription starts. Customer leaves the trial period with no commitment. Default
Before the end date: either end action can be triggered manually via API at any point during the trial. Conversion doesn't have to wait for the trial to expire.

Offering types supported

Not all offering types can be placed on a trial. Offerings must be recurring in nature.

Offering type Supported
Subscription Supported
Minimum commitment Supported
Token Supported
Custom percent of total Supported
One-time Not supported

At creation, MonetizeNow also validates that all offerings and rates are active, within their active date range, and that required products and quantities are correctly configured.

Trial states

A trial moves through a defined set of statuses from creation through to its final outcome. Only active trials can be actioned.

All statuses

Status What it means Terminal
ACTIVE Trial is running. It can be extended, manually converted, or cancelled.
AUTO_CONVERTED Batch job converted the trial at end date. Paid subscription is live. Quote ID is stored on the trial record. Final
MANUALLY_CONVERTED Converted via API before the end date. Same result as auto conversion: full quote, active subscription. Final
EXPIRED Trial ended with endAction = CANCEL. No subscription created. Final
MANUALLY_CANCELED Cancelled via API by an operator. Cancellation reason and timestamp are stored. Final

How states transition

Every trial starts at ACTIVE. From there, what happens next depends on either the endAction set at creation or a manual API call made during the trial period.

Trigger Condition Resulting status
End date reached (automatic) endAction = CANCEL EXPIRED
End date reached (automatic) endAction = CONVERT_TO_PAID AUTO_CONVERTED
Manual convert API call Trial is ACTIVE MANUALLY_CONVERTED
Manual cancel API call Trial is ACTIVE MANUALLY_CANCELED
Extend API call Trial is ACTIVE, new date is after current end date Stays ACTIVE with updated end date
All terminal statuses are final. A trial that has converted, expired, or been cancelled cannot be reopened.

Creating a trial

Trials are created via API. Everything needed to run the trial and convert it to a subscription is defined at creation time.

What you provide at creation

Two pieces of configuration drive the entire trial lifecycle.

Quote initialization details

Defines what the paid subscription quote will look like at conversion:

  • Contract length
  • Net terms
  • Primary contact
  • Billing contact

Conversion policy

Defines the products that go on the converted quote:

  • Which offerings to include
  • Which rate to apply per offering
  • Product IDs and quantities for each offering

Both are stored on the trial record and replayed automatically at conversion. They can't be changed after the trial is created.

Other required fields

Field Description
accountId Account the trial belongs to.
type NEW for a net-new trial. AMENDMENT is reserved for future use.
startDate When the trial period begins.
endDate When the trial period ends. Can be extended later while the trial is active.
endAction CONVERT_TO_PAID or CANCEL. Defaults to CANCEL if not specified.

Validation at creation

MonetizeNow validates all configuration before creating the trial. If anything doesn't pass, the trial is rejected and nothing is written.

  • All offerings must be active and within their active date range
  • All rates must be active and within their active date range
  • Account-scoped rates must belong to the same account as the trial
  • All required products on each rate must be included
  • Product quantities must satisfy the min and max rules on each rate
  • One-time offering types are not permitted

Converting to paid

When a trial converts, MonetizeNow builds and processes a full paid quote automatically using the configuration set at trial creation. No additional input is required.

How conversion works

Conversion can happen two ways: automatically when the trial's end date is reached and endAction = CONVERT_TO_PAID, or manually via the convert API endpoint at any point during the trial.

Either way, MonetizeNow runs the same sequence:

1
Build the quote
A net-new quote is created using the contract length, net terms, and contacts from the trial's quote initialization details. Contract start date is set to the current date.
2
Add offerings
Each entry in the trial's conversion policy is added to the quote as an offering, with the configured offering, rate, products, and quantities applied exactly as specified.
3
Process the quote
MonetizeNow automatically advances the quote through every stage: pending, review, approve, sent, accepted, and processed. No approvals need to be manually triggered.
4
Update the trial
Trial status updates to AUTO_CONVERTED or MANUALLY_CONVERTED. Quote ID is written to the trial record, linking the trial permanently to its resulting subscription.

When billing starts

No charges apply during the trial period. Billing starts from the contract start date on the subscription created at conversion, which is set to the date conversion runs.

Automatic conversion

A scheduled batch job runs periodically and picks up any trials where:

  • End date has passed
  • Status is ACTIVE
  • endAction is CONVERT_TO_PAID

Each trial is converted individually. If a batch run fails partway through, trials already converted keep their status. Nothing is rolled back.

USD only: converted quotes are always created in USD regardless of account settings. Multi-currency trial conversion is not yet supported.

Extending a trial

Active trials can be extended by updating the end date. This is useful when a customer needs more time to evaluate before committing.

How it works

Call the extend endpoint with a new end date. MonetizeNow updates the trial and the original end date is replaced. The trial continues as normal with the new expiry.

Two rules apply:

  • Only ACTIVE trials can be extended. Trials in any terminal status cannot be changed.
  • New end date must be after the current end date. You can't shorten a trial via extension.
Extension doesn't change the trial's endAction. A trial set to CONVERT_TO_PAID will still auto-convert at the new end date. A trial set to CANCEL will still expire.

Multiple extensions

Trials can be extended more than once. Each call to the extend endpoint replaces the current end date. There is no limit on how many times a trial can be extended as long as it stays active.

Canceling a trial

Trials can be cancelled at any point while active. Cancellation ends the evaluation period immediately without creating a subscription.

How it works

Call the cancel endpoint with a cancellation reason. MonetizeNow marks the trial as MANUALLY_CANCELED and stores the reason and cancellation timestamp on the trial record.

Only ACTIVE trials can be cancelled. Attempting to cancel a trial in a terminal status returns an error.

What cancellation does not do

Cancellation only affects the trial record. It does not:

  • Affect any other contracts or subscriptions on the account
  • Trigger any billing or credit actions
  • Prevent creating a new trial for the same account in future

Cancellation vs. expiry

Both result in no subscription being created, but they are different outcomes:

Outcome How it happens Status
Cancelled Operator calls the cancel API before end date MANUALLY_CANCELED
Expired End date passes with endAction = CANCEL EXPIRED

FAQs

Common questions about how trials work in MonetizeNow.

Can I change what products a trial converts onto after it's created?

No. Offering and product configuration in the conversion policy is locked at creation. To change what a customer converts onto, cancel the existing trial and create a new one with the updated configuration.

What happens if conversion fails?

If conversion fails mid-run, the trial stays ACTIVE. Nothing partial is written. Resolve the underlying issue - check that offerings and rates are still active, contacts are valid, and configuration is complete - then retry conversion.

Can a customer have more than one trial at the same time?

MonetizeNow doesn't enforce a limit on concurrent trials per account at the platform level. Whether to allow multiple simultaneous trials is a business decision to manage in your own logic.

Does converting a trial bypass the normal approval workflow?

Yes. Conversion automatically advances the quote through all stages including approval. No manual approvals are triggered. If your business requires an approval step before a trial converts, manage that process before calling the convert endpoint.

Can I extend a trial that has already expired or been cancelled?

No. Extension is only available on ACTIVE trials. Once a trial reaches any terminal status it can't be changed.

What currency are converted quotes created in?

Converted quotes are always created in USD. If your account uses a different currency, this is a current limitation. Multi-currency trial conversion is not yet supported.

Is there a way to set auto-renewal on the converted subscription?

Not currently via trial configuration. Converted quotes always have auto-renewal set to off. Auto-renewal can be configured on the resulting contract after conversion if needed.
